1. GOOGLE EARTH ENGINE — ACCESS & AUTH (required for reference imagery)
----------------------------------------------------------------
The GEE provider (src/dronomy_loc/reference/gee.py) needs a Google account
with Earth Engine access and a Cloud project. One-time setup:

[ ] 1.1  Sign up for Earth Engine (free, non-commercial):
         https://earthengine.google.com/  ->  "Get Started" / register a project.

[ ] 1.2  Create / pick a Google Cloud project and enable the Earth Engine API:
         https://console.cloud.google.com/  ->  APIs & Services -> enable
         "Google Earth Engine API". Note the PROJECT ID.

[ ] 1.3  Put the project id where the code can find it. Either:
           - edit config/config.yaml -> reference.gee.project: "your-project-id"
           - OR set an env var:   $env:EE_PROJECT = "your-project-id"   (PowerShell)

[ ] 1.4  Authenticate locally (interactive — opens a browser):
           earthengine authenticate
         (run it in the terminal: type  ! earthengine authenticate  in Claude,
          or run it yourself in PowerShell.)

[ ] 1.5  Smoke-test the fetch (writes data/reference/reference_gee.png):
           python scripts/02_fetch_reference.py
         Expect: saved raster size, ~m/px, and a center-pixel round-trip that
         matches (43.521955, -5.624290).

[!] 2.6  Earth Engine auth INSIDE Docker — pick one (then uncomment the matching
         volume line in docker/docker-compose.yml):
           (a) Mount your host EE credentials (simplest if you did step 1.4):
                 add to a .env file at repo root:
                   EE_PROJECT=your-project-id
                   EE_CREDENTIALS_DIR=C:/Users/Diego/.config/earthengine
                 and uncomment the "(a)" volume line in docker-compose.yml.
           (b) Headless service account (best for a VPS / CI):
                 - create a service account in Google Cloud, grant it Earth Engine,
                   download its JSON key,
                 - register the service account for Earth Engine use,
                 - in .env:
                     EE_PROJECT=your-project-id
                     EE_SERVICE_ACCOUNT=name@your-project.iam.gserviceaccount.com
                     EE_SERVICE_ACCOUNT_KEY=/run/secrets/ee-key.json
                     EE_KEY_FILE=C:/path/to/key.json   (host path)
                 - uncomment the "(b)" volume line in docker-compose.yml.
         (gee.py already supports both: service account if EE_SERVICE_ACCOUNT[_KEY]
          are set, otherwise stored user credentials.)
